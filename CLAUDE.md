## Project Overview

**OpenClaw Voice** is a native iOS app (Swift/SwiftUI) that lets users interact with OpenClaw (running on a Mac) via voice from an iPhone, including Apple CarPlay support. The system has two main pieces:

1. **iOS App** (iPhone + CarPlay) — captures voice, converts to text, sends commands, plays synthesized responses via ElevenLabs
2. **Relay Server** (Node.js on Mac) — bridges the iOS app and a local OpenClaw process over WebSocket

Communication is real-time bidirectional WebSocket (wss://) with JSON messages. The full protocol spec is in `docs/PROTOCOL.md`.

## Tech Stack

**iOS App:** Swift 5.9+, SwiftUI, iOS 17+, Apple Speech Framework (on-device STT), ElevenLabs REST API (streaming TTS), URLSessionWebSocketTask, AVAudioSession/AVAudioEngine, CarPlay Framework, Keychain Services

**Relay Server:** Node.js 20+, `ws` (WebSocket), `child_process` (OpenClaw process management), `jsonwebtoken`, `selfsigned` (TLS), `qrcode-terminal`

## Development Conventions

### Swift
- Use `@Observable` (not `ObservableObject`) and structured concurrency (`async/await`, not completion handlers)
- Use `guard` for early returns
- camelCase for variables/functions, PascalCase for types

### Node.js
- ESM imports (`import` not `require`)
- `async/await` with `try/catch`

### General Principles
- Native-only: no cross-platform frameworks
- Streaming-first: the entire pipeline (responses + TTS audio) must work with partial/chunked data
- CarPlay-ready from the start: design for two simultaneous interfaces (iPhone + CarPlay)
- Minimal dependencies: prefer Apple-native APIs; external packages only when they add clear value

## Relay Server Commands

```bash
cd openclaw-relay-server
npm install
node src/index.js

# Test WebSocket manually:
wscat -c wss://localhost:8765/ws --no-check
```

Config is stored at `~/.openclaw-relay/config.json`. TLS certs are auto-generated. Environment variables: `OPENCLAW_RELAY_PORT`, `OPENCLAW_COMMAND`, `OPENCLAW_WORKDIR`, `DEBUG`.

## iOS Project Setup

Xcode 15+, target iOS 17.0+, bundle ID `com.dckstudios.OpenClawVoice`. Required capabilities: Background Modes (Audio + VoIP), CarPlay (Communication). SPM dependencies: KeychainAccess, CodeScanner.

## Architecture Notes

- The relay server spawns OpenClaw as a child process and communicates via stdin/stdout. **Detecting end-of-response from OpenClaw is an open problem** — see `docs/SERVER.md` section 7.1 for strategies.
- iPhone and CarPlay share the same `AVAudioSession` — audio route transitions must be handled carefully.
- Authentication uses a token generated by the relay server on first run, shared via QR code or manual entry, stored in iOS Keychain.
- Remote access (outside local network) uses Tailscale VPN.
- The WebSocket protocol uses a streaming model: `response_start` → multiple `response_chunk` → `response_end`.
